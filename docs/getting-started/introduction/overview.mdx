# Overview

![Evaluation Journey](docs/assets/hero-evaluation-journey.svg)

## What is fasteval?

**fasteval** is a decorator-first LLM evaluation framework for Python. You stack decorators on test functions, run `pytest`, and get scored results. That's it.

Most teams treat eval as an afterthought -- something you bolt on once things start breaking in production. We built fasteval so evaluation fits directly into your dev loop, right next to your other tests.

```python
import fasteval as fe

@fe.correctness(threshold=0.8)
@fe.toxicity(threshold=0.95)
def test_my_agent():
    response = my_agent("How do I reset my password?")
    fe.score(response, "Navigate to Settings > Security > Reset Password")
```

Run `pytest` and fasteval handles everything: calling an LLM judge to assess quality, computing semantic scores, and failing tests when thresholds aren't met.

## Key Features

![Feature Cards](docs/assets/feature-cards.svg)

**Decorator-first API.** You define evaluation criteria directly on test functions. No config files, no boilerplate. Stack multiple decorators and you've got multi-metric evaluation out of the box.

**Pytest native.** It works with the test runner you already use -- full IDE integration, parallel execution, and CI/CD ready. We didn't want to invent a new runner. Pytest's the standard, so we built on it.

**LLM-as-Judge scoring.** Under the hood, fasteval uses an LLM judge for semantic evaluation. It understands *meaning*, not just string matching. In practice, this catches issues that regex-based checks miss entirely.

**20+ built-in metrics** covering correctness, toxicity, hallucination, faithfulness, and more. And if the built-ins don't fit your use case, you can build custom metrics tailored to your domain.

**Production monitoring.** The `fasteval-observe` plugin lets you evaluate live systems with configurable sampling -- so you're not just testing in CI, you're watching prod too.

### Who's this for?

If you're building anything with LLMs and need to know whether it's actually working, fasteval's probably useful. That could mean setting up automated quality gates for an LLM-powered feature, establishing eval standards across a platform with multiple AI apps, running structured experiments on prompt variations, or extending test coverage to non-deterministic outputs that traditional assertions can't handle.

## See It In Action

### Basic Quality Check

```python
@fe.correctness(threshold=0.7)
def test_qa_accuracy():
    """Verify our QA bot gives accurate answers."""
    response = qa_bot("What year was Python released?")
    fe.score(response, expected_output="1991")
```

### Multi-Metric Evaluation

```python
@fe.correctness(threshold=0.8)
@fe.relevance(threshold=0.7)
@fe.toxicity(threshold=0.95)
def test_customer_support():
    """Evaluate support agent across multiple quality dimensions."""
    response = support_agent("I want a refund!")
    fe.score(response, expected_output="I understand your concern...")
```

### RAG Pipeline Evaluation

```python
@fe.faithfulness(threshold=0.85)
@fe.contextual_precision(threshold=0.7)
def test_rag_grounding():
    """Ensure answers are grounded in retrieved context."""
    docs = retriever.search("company policy")
    response = rag_agent("What is the return policy?", context=docs)
    fe.score(response, context=docs)
```

### Production Monitoring

```python
from fasteval_observe import observe
from fasteval_observe.sampling import FixedRateSamplingStrategy

@observe(sampling=FixedRateSamplingStrategy(rate=0.05))
@fe.toxicity(threshold=0.95)
async def production_chatbot(query: str) -> str:
    """Monitor 5% of production traffic for safety."""
    return await llm.invoke(query)
```
