# Vision Metrics

These metrics are built for vision-language models (VLMs) — systems that take images plus text and produce text responses.

## Installation

```bash
pip install fasteval-core[vision]
```

## Available Metrics

### `@fe.image_understanding`

Does the model actually understand what's in the image? This metric checks.

```python
import fasteval as fe

@fe.image_understanding(threshold=0.8)
def test_vision_qa():
    response = gpt4v.chat(
        image="chart.png",
        prompt="What trend does this chart show?"
    )
    fe.score(
        actual_output=response,
        expected_output="Revenue increased 25% year over year",
        image="chart.png",
        input="What trend does this chart show?"
    )
```

**Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `threshold` | float | 0.7 | Minimum score to pass |
| `weight` | float | 1.0 | Weight for aggregate scoring |
| `model` | str | None | Vision model override |

### `@fe.ocr_accuracy`

Evaluates text extraction accuracy from images and documents. It supports two modes: field-based extraction using `expected_fields` for comparing specific values, and full text comparison using `expected_output` for the entire extracted text.

```python
@fe.ocr_accuracy(threshold=0.95)
def test_document_extraction():
    response = vision_model.extract("invoice.pdf")
    fe.score(
        actual_output=response,
        expected_fields={"total": "$1,234.56", "date": "2024-01-15"}
    )
```

**Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `threshold` | float | 0.9 | Minimum accuracy to pass |
| `weight` | float | 1.0 | Weight for aggregate scoring |

### `@fe.chart_interpretation`

Evaluates correct reading of charts, graphs, and data visualizations. It checks numerical values, trend and pattern identification, and whether labels, legends, and axes are interpreted correctly.

```python
@fe.chart_interpretation(threshold=0.8)
def test_chart_reading():
    response = model.analyze(
        image="sales_chart.png",
        query="What was Q3 revenue?"
    )
    fe.score(
        actual_output=response,
        expected_output="$4.2 million",
        image="sales_chart.png",
        input="What was Q3 revenue?"
    )
```

### `@fe.visual_grounding`

Can the model point to the right place? This metric evaluates whether responses correctly reference specific image regions — object identification, spatial references (left, right, top, bottom), and relative positions between objects.

```python
@fe.visual_grounding(threshold=0.8)
def test_object_location():
    response = model.locate(
        image="room.jpg",
        query="Where is the lamp?"
    )
    fe.score(
        actual_output=response,
        expected_output="The lamp is on the desk in the upper right corner",
        image="room.jpg"
    )
```

### `@fe.image_faithfulness`

Catches hallucinations. This metric verifies that every claim in the response is actually visible in the image — no hallucinated details, no misattributions.

```python
@fe.image_faithfulness(threshold=0.85)
def test_faithful_description():
    response = model.describe(image="photo.jpg")
    fe.score(
        actual_output=response,
        image="photo.jpg"
    )
```

## Image Input Formats

All vision metrics accept images in multiple formats:

```python
# File path (string)
fe.score(response, image="path/to/image.png")

# URL
fe.score(response, image="https://example.com/image.jpg")

# ImageInput model
from fasteval import ImageInput
fe.score(response, image=ImageInput(source="image.png", alt_text="Sales chart"))

# Multiple images
fe.score(response, images=["img1.png", "img2.png", "img3.png"])
```

## Custom Vision Metrics

Create custom vision metrics by extending `BaseVisionMetric`:

```python
from fasteval.metrics.vision import BaseVisionMetric
from fasteval.models.evaluation import EvalInput, MetricResult

class MyVisionMetric(BaseVisionMetric):
    def __init__(self, **kwargs):
        kwargs.setdefault("name", "my_vision_metric")
        super().__init__(**kwargs)

    def get_evaluation_prompt(self, eval_input: EvalInput) -> str:
        return f"""Evaluate the image response.

Query: {eval_input.input}
Response: {eval_input.actual_output}

Provide your evaluation as JSON:
{{"score": <0.0 to 1.0>, "reasoning": "<explanation>"}}"""

# Use with @fe.metric decorator
@fe.metric(MyVisionMetric(threshold=0.8))
def test_custom():
    response = model.analyze(image="test.png")
    fe.score(response, image="test.png")
```

## Best Practices

Make sure you're using a vision-capable model — set `model="gpt-4-vision-preview"` or similar. Large images get automatically resized to max 2048px, but it's still worth keeping input sizes reasonable. Always include the original query in `input` so the evaluator has proper context. For extraction tasks, `expected_fields` gives you much more precise results than full text comparison.
