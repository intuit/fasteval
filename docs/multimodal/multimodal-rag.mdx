# Multi-Modal RAG Metrics

When your RAG system handles documents with text, tables, images, and figures all mixed together, you need metrics that understand every modality. That's what these are for.

## Installation

```bash
pip install fasteval-core[vision]
```

## Available Metrics

### `@fe.multimodal_faithfulness`

Evaluates whether the response stays faithful to multi-modal context — both text and images. It checks that claims are supported by the text context, supported by visual content, and that nothing unsupported sneaks in from either modality.

```python
import fasteval as fe
from fasteval import ImageInput

@fe.multimodal_faithfulness(threshold=0.8)
def test_document_qa():
    response = rag_system.query(
        question="What was Q3 revenue?",
        documents=["financial_report.pdf"]
    )
    fe.score(
        actual_output=response,
        context=retrieved_text_chunks,
        images=[ImageInput(source="chart_page_5.png")],
        expected="$4.2M",
        input="What was Q3 revenue?"
    )
```

### `@fe.table_extraction`

Measures accuracy when extracting table data from document images.

```python
@fe.table_extraction(threshold=0.9)
def test_table_qa():
    response = model.answer(
        image="financial_table.png",
        question="What is the total for 2024?"
    )
    fe.score(
        actual_output=response,
        expected="$150,000",
        image="financial_table.png",
        input="What is the total for 2024?"
    )
```

**With expected fields:**

```python
@fe.table_extraction(threshold=0.9)
def test_table_extraction():
    extracted = model.extract_table("invoice.png")
    fe.score(
        actual_output=extracted,
        image="invoice.png",
        expected_fields={
            "subtotal": "$1,000.00",
            "tax": "$80.00",
            "total": "$1,080.00",
            "date": "2024-01-15"
        }
    )
```

### `@fe.figure_reference`

Checks whether the response correctly references figures and charts. It verifies that references are accurate, figure numbers and labels are attributed properly, and data cited from figures is correct.

```python
@fe.figure_reference(threshold=0.8)
def test_figure_citation():
    response = model.analyze(
        images=["doc_with_figures.png"],
        query="Summarize the trends shown in the figures"
    )
    fe.score(
        actual_output=response,
        images=["doc_with_figures.png"],
        input="Summarize the trends shown in the figures"
    )
```

### `@fe.cross_modal_coherence`

Evaluates coherence between text and visual elements. Does the text accurately describe what's in the image? Are there contradictions? Is the style consistent across modalities?

```python
@fe.cross_modal_coherence(threshold=0.8)
def test_caption_coherence():
    caption = model.caption(image="photo.jpg")
    fe.score(
        actual_output=caption,
        image="photo.jpg"
    )
```

### `@fe.document_understanding`

Evaluates how well the model grasps complex documents with mixed content types — correct interpretation of document structure, accurate reading of tables/forms/figures, and integration of context from multiple sections.

```python
@fe.document_understanding(threshold=0.8)
def test_contract_qa():
    response = model.query(
        document="contract.pdf",
        question="What is the termination clause?"
    )
    fe.score(
        actual_output=response,
        expected_output="30-day written notice",
        image="contract.pdf",  # or list of page images
        input="What is the termination clause?"
    )
```

## Complete Multi-Modal RAG Example

```python
import fasteval as fe
from fasteval import ImageInput, MultiModalContext

@fe.multimodal_faithfulness(threshold=0.8)
@fe.table_extraction(threshold=0.9)
@fe.figure_reference(threshold=0.8)
def test_financial_report_qa():
    """Evaluate QA over a financial report with charts and tables."""

    query = "Compare Q3 and Q4 revenue and explain the trend"

    # Retrieve relevant content
    text_chunks = retriever.get_text_chunks(query, doc="report.pdf")
    relevant_images = retriever.get_images(query, doc="report.pdf")

    # Generate response
    response = llm.generate(
        query=query,
        context=text_chunks,
        images=relevant_images
    )

    fe.score(
        actual_output=response,
        context=text_chunks,
        images=relevant_images,
        input=query,
        expected_output="Q4 revenue was $5.2M, up 24% from Q3's $4.2M, driven by holiday sales"
    )
```

## Working with Multi-Modal Context

### Using MultiModalContext

```python
from fasteval import MultiModalContext, ImageInput

# Create rich context
context = MultiModalContext(
    text="The quarterly report shows strong growth...",
    images=[
        ImageInput(source="chart.png", alt_text="Revenue Chart"),
        ImageInput(source="table.png", alt_text="Financial Table")
    ],
    metadata={"page_range": "5-8"}
)

fe.score(
    actual_output=response,
    multimodal_context=[context],
    input=query
)
```

### Multiple Page Documents

```python
# Process multi-page PDFs
page_images = [
    ImageInput(source=f"page_{i}.png", metadata={"page": i})
    for i in range(1, 11)
]

fe.score(
    actual_output=response,
    images=page_images,
    context=text_chunks,
    input=query
)
```

## Integration with RAG Pipelines

### LangChain Integration

```python
from langchain.chains import RetrievalQA
from langchain.vectorstores import Chroma

@fe.multimodal_faithfulness(threshold=0.8)
def test_langchain_rag():
    # Your LangChain RAG setup
    qa_chain = RetrievalQA.from_chain_type(...)

    result = qa_chain.run(query)
    retrieved_docs = retriever.get_relevant_documents(query)

    fe.score(
        actual_output=result,
        context=[doc.page_content for doc in retrieved_docs],
        images=[doc.metadata.get("image") for doc in retrieved_docs if doc.metadata.get("image")],
        input=query
    )
```

## Best Practices

Multi-modal faithfulness needs context from both modalities, so always include text and images together. Convert PDF pages to images for visual context — page-level images work best. When evaluating table extraction, use `expected_fields` instead of full text comparison for more precise results. Make sure to test queries that span multiple tables or figures, since those are the cases most likely to break. And include page numbers in your metadata; you'll thank yourself when debugging.
