# Audio Metrics

Got a speech-to-text pipeline, voice assistant, or audio understanding system? Here's how to evaluate it.

## Installation

```bash
pip install fasteval-core[audio]
```

## Available Metrics

### `@fe.word_error_rate`

Word Error Rate (WER) is the industry standard for ASR. It measures the minimum word-level edits needed to transform the hypothesis into the reference.

```python
import fasteval as fe

@fe.word_error_rate(threshold=0.95)  # WER < 5%
def test_whisper():
    transcript = whisper.transcribe("audio.mp3")
    fe.score(
        actual_output=transcript,
        expected_output=ground_truth_transcript
    )
```

The score is inverted (1.0 - WER) so higher is better. A WER of 5% gives a score of 0.95, meaning a threshold of 0.95 requires WER under 5%.

**Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `threshold` | float | 0.9 | Minimum accuracy (1.0 - WER) to pass |
| `normalize_text` | bool | True | Normalize text before comparison |

### `@fe.character_error_rate`

Character Error Rate (CER) gives you finer-grained accuracy analysis. It's especially useful for languages without clear word boundaries like Chinese or Japanese, for name/entity recognition, or when exact character accuracy matters.

```python
@fe.character_error_rate(threshold=0.95)
def test_asr_detailed():
    transcript = model.transcribe(audio)
    fe.score(
        actual_output=transcript,
        expected_output=reference
    )
```

### `@fe.match_error_rate`

Match Error Rate (MER) is a WER variant that's always bounded between 0 and 1.

```python
@fe.match_error_rate(threshold=0.9)
def test_asr():
    transcript = model.transcribe(audio)
    fe.score(
        actual_output=transcript,
        expected_output=reference
    )
```

### `@fe.transcription_accuracy`

This one uses an LLM to evaluate semantic accuracy rather than exact word matching. Unlike WER/CER, it doesn't penalize you for minor word variations ("gonna" vs "going to"), different number formats ("5" vs "five"), or synonyms that carry the same meaning.

```python
@fe.transcription_accuracy(threshold=0.9)
def test_semantic_accuracy():
    transcript = whisper.transcribe("meeting.mp3")
    fe.score(
        actual_output=transcript,
        expected_output=ground_truth_transcript
    )
```

### `@fe.speaker_diarization`

Evaluates speaker identification accuracy in multi-speaker transcriptions. It checks whether speakers are correctly identified, turn boundaries are accurate, and labeling stays consistent throughout.

```python
@fe.speaker_diarization(threshold=0.8)
def test_speaker_id():
    result = model.transcribe_with_speakers("meeting.mp3")
    fe.score(
        actual_output=result,
        expected_output=annotated_transcript
    )
```

### `@fe.audio_sentiment`

Evaluates tone and emotion detection accuracy from audio.

```python
@fe.audio_sentiment(threshold=0.8)
def test_tone_detection():
    sentiment = model.detect_sentiment("customer_call.mp3")
    fe.score(
        actual_output=sentiment,
        expected_output="frustrated, but cooperative"
    )
```

## WER vs CER vs MER

| Metric | Use Case | Bounded |
|--------|----------|---------|
| **WER** | General ASR evaluation | No (can exceed 1.0) |
| **CER** | Character-level accuracy, CJK languages | No |
| **MER** | When you need bounded 0-1 score | Yes (0-1) |

## Best Practices

We recommend enabling `normalize_text=True` (the default) to ignore case and punctuation differences. Pick the right metric for the job: WER for general ASR, CER for character-level work. In practice, combining deterministic and LLM metrics works best — use WER for exact accuracy and `transcription_accuracy` for semantic correctness. For thresholds, industry-standard WER sits around 5% for clean audio, but you'll want to be more lenient for noisy or challenging conditions.
