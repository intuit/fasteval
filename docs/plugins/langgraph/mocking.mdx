# Node Mocking

Sometimes you need to swap out a real node for a fake. Maybe the RAG service is slow, or you want to feed exact state to a downstream node. That's what mocking is for.

## Overview

Node mocking lets you isolate the node under test by replacing its dependencies with controlled fakes. In practice, we use this to test the planner without calling the real RAG service, feed specific state to downstream nodes, verify routing with `.goto()`, and skip slow LLM calls in upstream nodes.

## Fluent Builder API

The `mock()` function returns a `NodeMock` builder with chainable methods:

```python
from fasteval_langgraph import mock

# Static updates + routing
mock("rag").updates({"docs": ["fake doc"]}).goto("planner")

# Just state updates (no routing override)
mock("logger").updates({"log": "captured"})

# Just routing (no state updates)
mock("router").goto("fallback")
```

### NodeMock Methods

| Method | Description |
|--------|-------------|
| `.updates(dict)` | Static state updates the mock returns |
| `.goto(str)` | Routing target (produces `Command(goto=...)`) |
| `.fn(callable)` | Dynamic handler `(state) -> dict` |

All methods return `self` for chaining.

## Mocking Patterns

### In-Place (mutate the harness)

```python
graph.mock("rag").updates({"docs": ["fake"]}).goto("responder")
result = await graph.chat("test")
graph.reset_mocks()  # restore originals
```

### Immutable Copy (safe for parallel tests)

```python
test_graph = graph.with_mocks(
    mock("rag").updates({"docs": ["fake"]}).goto("responder"),
    mock("classifier").updates({"intent": "FAQ"}).goto("rag"),
)
result = await test_graph.chat("test")
# Original graph is unaffected
```

### Scoped Context Manager (auto-restore)

```python
with graph.mocked(
    mock("rag").updates({"docs": ["fake"]}).goto("responder"),
):
    result = await graph.chat("test")
    assert result.state["docs"] == ["fake"]

# Mocks automatically restored after the with block
result2 = await graph.chat("test")
assert result2.state["docs"] != ["fake"]  # real node runs
```

## Dynamic Mocks

Use `.fn()` when your mock needs to read state:

```python
with graph.mocked(
    mock("rag").fn(
        lambda state: {"docs": [f"doc for {state.get('intent', '?')}"]}
    ).goto("responder"),
):
    result = await graph.chat("What is OAuth?")
    assert "doc for FAQ" in result.state["docs"][0]
```

The handler receives the current state dict and should return a dict of state updates. If you need both updates and routing, return `{"updates": {...}, "goto": "..."}`:

```python
mock("smart_router").fn(lambda state: {
    "updates": {"routed": True},
    "goto": "planner" if state.get("complex") else "responder",
})
```

## Combining with fasteval

Mock upstream nodes, then evaluate the node under test with fasteval decorators:

```python
import fasteval as fe
from fasteval_langgraph import harness, mock

graph = harness(compiled_graph)

@fe.criteria("Does the planner create a multi-path diagnostic plan?")
@pytest.mark.asyncio
async def test_planner_with_mock_rag():
    test_graph = graph.with_mocks(
        mock("classifier").updates({"intent": {"category": "TROUBLESHOOTING"}}).goto("rag"),
        mock("rag").updates({"docs": ["tracing documentation..."]}).goto("planner"),
    )
    result = await test_graph.chat("My traces are missing")
    fe.score(
        str(result.state.get("plan")),
        "Multi-path diagnostic plan for missing traces",
        input="My traces are missing",
    )
    test_graph.reset_mocks()
```
