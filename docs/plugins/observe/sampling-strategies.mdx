# Sampling Strategies

Which calls get observed? That's what sampling strategies decide. Pick one based on your monitoring goals, latency budget, and resource constraints.

## Overview

| Strategy | Use Case | Overhead | Deterministic |
|----------|----------|----------|---------------|
| NoSampling | Development | High | Yes |
| FixedRate | Production steady-state | Low | Yes |
| Probabilistic | Statistical profiling | Low | No |
| Adaptive | Incident response | Medium | No |
| TokenBudget | Cost monitoring | Low | No |
| Composable | Complex scenarios | Varies | Depends |

## NoSamplingStrategy

Samples every single call. Use it for development, debugging, or critical paths where you can't afford to miss anything.

```python
from fasteval_observe import observe
from fasteval_observe.sampling import NoSamplingStrategy

@observe(sampling=NoSamplingStrategy())
async def debug_agent(query: str) -> str:
    return await llm.invoke(query)
```

Good for local development, debugging issues, critical paths that must always be monitored, and low-traffic functions. Don't use it on high-traffic production endpoints or when latency is critical.

## FixedRateSamplingStrategy

Samples at a fixed rate using an internal counter. It's deterministic -- the same Nth calls always get sampled.

```python
from fasteval_observe.sampling import FixedRateSamplingStrategy

# Sample 1 in 100 calls (1%)
@observe(sampling=FixedRateSamplingStrategy(rate=0.01))
async def my_agent(query: str) -> str:
    return await llm.invoke(query)

# Sample 1 in 20 calls (5%)
@observe(sampling=FixedRateSamplingStrategy(rate=0.05))
async def higher_rate_agent(query: str) -> str:
    return await llm.invoke(query)
```

Takes one parameter: `rate` (float between 0.0 and 1.0). Internally it samples every Nth call where N = 1/rate. We recommend this as the default for production monitoring -- it gives you predictable resource usage and a consistent sampling rate.

## ProbabilisticSamplingStrategy

Random probability-based sampling. Non-deterministic, but you can pin it with a seed if you need reproducibility.

```python
from fasteval_observe.sampling import ProbabilisticSamplingStrategy

# 5% probability each call
@observe(sampling=ProbabilisticSamplingStrategy(probability=0.05))
async def my_agent(query: str) -> str:
    return await llm.invoke(query)

# Reproducible with seed
@observe(sampling=ProbabilisticSamplingStrategy(probability=0.1, seed=42))
async def reproducible_agent(query: str) -> str:
    return await llm.invoke(query)
```

Parameters: `probability` (float, 0.0-1.0) and an optional `seed` (int) for reproducibility. Best for statistical profiling, A/B testing observability, or when the exact sampling rate doesn't matter.

## AdaptiveSamplingStrategy

This one gets interesting. It dynamically adjusts the sampling rate based on latency and error patterns.

```python
from fasteval_observe.sampling import AdaptiveSamplingStrategy

@observe(sampling=AdaptiveSamplingStrategy(
    base_rate=0.01,           # 1% baseline
    error_rate=1.0,           # 100% on errors
    slow_threshold_ms=2000,   # What counts as "slow"
    slow_rate=0.5,            # 50% of slow calls
    window_size=100,          # Look at last 100 calls
    cooldown_seconds=60.0,    # Stay elevated for 60s after incident
))
async def my_agent(query: str) -> str:
    return await llm.invoke(query)
```

**Parameters:**

- `base_rate` -- Normal operation sampling rate
- `error_rate` -- Rate when errors are detected (default: 1.0)
- `slow_threshold_ms` -- Latency threshold for "slow" calls
- `slow_rate` -- Rate for slow calls
- `window_size` -- How many recent calls to consider

Here's how it works: the strategy tracks recent latencies and errors. If more than 10% of calls in the window are errors, it switches to `error_rate`. If more than 10% are slow, it uses `slow_rate`. After an incident subsides, it stays elevated for `cooldown_seconds` before dropping back to `base_rate`.

In practice, this is the strategy we reach for when things go wrong in production.

## TokenBudgetSamplingStrategy

Samples based on token usage and cost. Useful when you need to keep evaluation spend under control.

```python
from fasteval_observe.sampling import TokenBudgetSamplingStrategy

@observe(sampling=TokenBudgetSamplingStrategy(
    budget_tokens_per_hour=100000,  # Token budget
    high_cost_threshold=1000,       # "High cost" = >1000 tokens
    high_cost_rate=0.5,             # Sample 50% of high-cost
    base_rate=0.05,                 # Otherwise 5%
))
async def my_agent(query: str) -> str:
    return await llm.invoke(query)
```

It tracks hourly token usage from completion callbacks. High-cost calls (based on context) get sampled at `high_cost_rate`. As you approach your budget -- past 70% usage, the rate increases. Past 90%, it jumps to 5x the base rate. Otherwise it sticks to `base_rate`.

## ComposableSamplingStrategy

Combine multiple strategies with AND/OR logic when a single strategy doesn't cut it.

```python
from fasteval_observe.sampling import (
    ComposableSamplingStrategy,
    ProbabilisticSamplingStrategy,
    AdaptiveSamplingStrategy,
)

# OR logic: sample if ANY strategy says yes
@observe(sampling=ComposableSamplingStrategy(
    strategies=[
        ProbabilisticSamplingStrategy(probability=0.01),  # 1% random
        AdaptiveSamplingStrategy(base_rate=0.0, error_rate=1.0),  # All errors
    ],
    mode="any"  # OR logic
))
async def my_agent(query: str) -> str:
    return await llm.invoke(query)

# AND logic: sample only if ALL strategies agree
@observe(sampling=ComposableSamplingStrategy(
    strategies=[
        FixedRateSamplingStrategy(rate=0.1),
        TokenBudgetSamplingStrategy(budget_tokens_per_hour=50000),
    ],
    mode="all"  # AND logic
))
async def conservative_agent(query: str) -> str:
    return await llm.invoke(query)
```

Two parameters: `strategies` (list of strategies to combine) and `mode` (`"any"` for OR, `"all"` for AND).

### Quick Reference

| Scenario | Recommended Strategy |
|----------|---------------------|
| Local development | NoSamplingStrategy |
| Production monitoring | FixedRateSamplingStrategy(rate=0.01) |
| Statistical profiling | ProbabilisticSamplingStrategy |
| Auto-scale on incidents | AdaptiveSamplingStrategy |
| Cost tracking | TokenBudgetSamplingStrategy |
| Sample errors + 1% random | ComposableSamplingStrategy(mode="any") |
