# Sampling Strategies

## The Economics of Evaluation

Every evaluation calls a judge LLM, and those calls add up. Take a typical setup: 10,000 traces/day, 3 metrics each, at ~$0.003 per call. That's $90/day, $2,700/month. Not great.

With 2% random sampling (200 traces/day), you're down to $1.80/day and $54/month — a 98% cost reduction while still getting statistically valid quality measurements.

Sampling makes sense when you're doing ongoing monitoring at scale and your budget is finite. It doesn't make sense when regulations require evaluating every trace, volume is already low (under 100/day), or you're investigating a specific incident and need the full picture.

## Sampling Fundamentals

### How Sampling Works

When you specify a sampling strategy, the flow is: fetch all matching traces from Langfuse, let the strategy select a subset, evaluate only those, and report results for the sampled population.

```python
from fasteval_langfuse import langfuse_traces
from fasteval_langfuse.sampling import RandomSamplingStrategy

@langfuse_traces(
    project="production",
    time_range="last_24h",
    sampling=RandomSamplingStrategy(sample_size=200)  # Evaluate 200 traces
)
def test_sampled(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

### Sample Size vs. Confidence

Bigger samples give more confidence but cost more:

| Sample Size | Margin of Error (95% confidence) | Relative Cost |
|-------------|----------------------------------|---------------|
| 50 | ±14% | Low |
| 100 | ±10% | Low |
| 200 | ±7% | Medium |
| 400 | ±5% | Medium |
| 1000 | ±3% | High |

For most monitoring, 100-400 samples hit the sweet spot between confidence and cost.

### Sampling Bias

Not all approaches are equal. If you only sample peak-hour traces, you'll miss overnight interactions from international users or automated systems. The built-in strategies address common bias concerns, but it's worth thinking about what your sampling might miss.

## Built-in Strategies

Five strategies ship with the plugin.

### RandomSamplingStrategy

Every trace has an equal shot at selection. Simple, unbiased, no assumptions about trace distribution.

```python
from fasteval_langfuse.sampling import RandomSamplingStrategy

@langfuse_traces(
    project="production",
    sampling=RandomSamplingStrategy(
        sample_size=200,    # Number of traces to select
        seed=42             # Optional: for reproducible sampling
    )
)
def test_random_sample(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `sample_size` | `int` | Number of traces to sample |
| `seed` | `int` (optional) | Random seed for reproducibility |

Best for general quality monitoring, baseline measurements, and when you don't have prior information about trace quality. One downside: it may under-represent rare categories.

### StratifiedSamplingStrategy

Samples equally from each group, ensuring representation across categories.

```python
from fasteval_langfuse.sampling import StratifiedSamplingStrategy

@langfuse_traces(
    project="production",
    sampling=StratifiedSamplingStrategy(
        strata_key="metadata.user_type",  # Group by user type
        samples_per_stratum=50,           # 50 samples from each group
        seed=42
    )
)
def test_stratified(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `strata_key` | `str` | Key to group by (supports dot notation) |
| `samples_per_stratum` | `int` | Samples from each group |
| `seed` | `int` (optional) | Random seed |

Use this when you need fair quality measurement across user segments or when some categories are rare but important.

```python
# By user segment
StratifiedSamplingStrategy(strata_key="metadata.user_type", samples_per_stratum=50)
# Groups: "enterprise", "pro", "free" → 150 total samples

# By query complexity
StratifiedSamplingStrategy(strata_key="metadata.complexity", samples_per_stratum=30)
# Groups: "simple", "moderate", "complex" → 90 total samples

# By region
StratifiedSamplingStrategy(strata_key="metadata.region", samples_per_stratum=40)
# Groups: "us-east", "us-west", "eu", "asia" → 160 total samples
```

### ScoreBasedSamplingStrategy

Oversamples low-quality traces based on existing scores. Perfect for debugging.

```python
from fasteval_langfuse.sampling import ScoreBasedSamplingStrategy

@langfuse_traces(
    project="production",
    sampling=ScoreBasedSamplingStrategy(
        score_name="user_rating",      # Existing score in Langfuse
        low_score_threshold=3.0,       # Scores ≤ 3.0 are "low"
        low_score_rate=1.0,            # Sample 100% of low scores
        high_score_rate=0.05,          # Sample 5% of high scores
        seed=42
    )
)
def test_score_based(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `score_name` | `str` | Name of existing score to filter on |
| `low_score_threshold` | `float` | Threshold below which scores are "low" |
| `low_score_rate` | `float` | Sampling rate for low scores (0.0-1.0) |
| `high_score_rate` | `float` | Sampling rate for high scores (0.0-1.0) |
| `seed` | `int` (optional) | Random seed |

Use it for root cause analysis, focusing evaluation budget on failures, or when user ratings already exist.

```python
# Debug mode: evaluate all failures, sample successes
ScoreBasedSamplingStrategy(
    score_name="user_thumbs_up",
    low_score_threshold=0,       # 0 = thumbs down
    low_score_rate=1.0,          # All thumbs-down traces
    high_score_rate=0.1          # 10% of thumbs-up traces
)

# Quality investigation: focus on borderline cases
ScoreBasedSamplingStrategy(
    score_name="fasteval_correctness",
    low_score_threshold=0.7,
    low_score_rate=0.8,          # 80% of low scores
    high_score_rate=0.2          # 20% of high scores
)
```

### RecentFirstSamplingStrategy

Weighted toward newer traces using exponential decay. Recent traces get higher selection probability.

```python
from fasteval_langfuse.sampling import RecentFirstSamplingStrategy

@langfuse_traces(
    project="production",
    time_range="last_30d",
    sampling=RecentFirstSamplingStrategy(
        sample_size=500,
        decay_factor=0.8,    # Higher = more recent bias
        seed=42
    )
)
def test_recent_bias(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `sample_size` | `int` | Number of traces to sample |
| `decay_factor` | `float` | Decay rate (0.0-1.0); higher = more recent bias |
| `seed` | `int` (optional) | Random seed |

Good for post-deployment validation, detecting recent quality changes, or when newer data matters more. A decay factor of 0.5 creates strong recent bias, 0.8 is moderate, and 0.95 is nearly uniform.

### NoSamplingStrategy

Evaluates everything. No selection, no randomness.

```python
from fasteval_langfuse.sampling import NoSamplingStrategy

@langfuse_traces(
    project="production",
    sampling=NoSamplingStrategy()  # Evaluate everything
)
def test_all_traces(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

Use when volume is low (under 100 traces), for incident investigations where you need the complete picture, or when regulations require it.

## Choosing a Strategy

| Your Situation | Recommended Strategy |
|----------------|---------------------|
| General monitoring, no special requirements | RandomSamplingStrategy |
| Need quality by user segment | StratifiedSamplingStrategy |
| Debugging failures, have user ratings | ScoreBasedSamplingStrategy |
| Focus on recent behavior | RecentFirstSamplingStrategy |
| Low volume or audit requirement | NoSamplingStrategy |

### Combining Strategies with Filters

Narrow your population with filters first, then sample the remainder:

```python
@langfuse_traces(
    project="production",
    filter_tags=["customer-support"],  # Filter first
    time_range="last_7d",              # Narrow time window
    sampling=RandomSamplingStrategy(sample_size=200)  # Then sample
)
def test_support_quality(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

### Multiple Evaluations with Different Strategies

Run parallel tests for different purposes:

```python
# General monitoring: random sample
@langfuse_traces(
    project="production",
    time_range="last_24h",
    sampling=RandomSamplingStrategy(sample_size=200)
)
def test_daily_quality(trace_id, input, output, context, metadata):
    fe.score(output, input=input)


# Failure analysis: oversample low quality
@langfuse_traces(
    project="production",
    time_range="last_24h",
    sampling=ScoreBasedSamplingStrategy(
        score_name="user_rating",
        low_score_threshold=3.0,
        low_score_rate=1.0,
        high_score_rate=0.0
    )
)
def test_failures_only(trace_id, input, output, context, metadata):
    fe.score(output, input=input)


# Segment analysis: stratified by user type
@langfuse_traces(
    project="production",
    time_range="last_24h",
    sampling=StratifiedSamplingStrategy(
        strata_key="metadata.user_type",
        samples_per_stratum=50
    )
)
def test_by_segment(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

## Custom Strategies

Need something the built-in strategies don't cover? Extend `BaseSamplingStrategy`:

```python
from typing import Any, Dict, List
from fasteval_langfuse.sampling.base import BaseSamplingStrategy

class TimeOfDaySamplingStrategy(BaseSamplingStrategy):
    """
    Sample traces proportionally across hours of the day.

    Catches issues that only show up at specific times.
    """

    def __init__(self, samples_per_hour: int = 10, seed: int = None):
        self.samples_per_hour = samples_per_hour
        self.seed = seed
        self._rng = random.Random(seed)

    def sample(self, traces: List[Dict[str, Any]]) -> List[Dict[str, Any]]:
        # Group traces by hour
        by_hour: Dict[int, List[Dict[str, Any]]] = {}

        for trace in traces:
            timestamp = trace.get("timestamp")
            if timestamp:
                hour = parse_hour(timestamp)
                by_hour.setdefault(hour, []).append(trace)

        # Sample from each hour
        sampled = []
        for hour, hour_traces in by_hour.items():
            sample_size = min(self.samples_per_hour, len(hour_traces))
            sampled.extend(self._rng.sample(hour_traces, sample_size))

        return sampled
```

Use it like any built-in strategy:

```python
@langfuse_traces(
    project="production",
    time_range="last_24h",
    sampling=TimeOfDaySamplingStrategy(samples_per_hour=15)
)
def test_hourly_coverage(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

## Statistical Considerations

### Sample Size Selection

The right sample size depends on population size, desired confidence level, expected variability, and acceptable margin of error. As a rule of thumb for monitoring where ±10% is fine: 100-200 samples for populations up to 10,000, and 200-400 for larger ones.

### Interpreting Sampled Results

Aggregate scores are estimates, not exact values. A correctness score of 0.85 from 100 samples means the true value is probably between 0.78 and 0.92 (at 95% confidence).

Watch trends over time rather than fixating on single runs. A drop from 0.85 to 0.82 in one run might be noise. Five consecutive declines aren't.

Be careful with segment-level results too. If you sampled 200 traces but only 20 are from one segment, that segment's estimate has high uncertainty.

### Handling Low-Volume Periods

During low-traffic periods, your requested sample size may exceed available traces. The plugin handles this gracefully — if you request 200 but only 50 exist, you get all 50.

### Reproducibility

For debugging or comparison, use a seed:

```python
# Same seed produces same sample
sampling=RandomSamplingStrategy(sample_size=200, seed=42)
```

Reproducibility requires the same seed, same underlying trace population, and same strategy config.
