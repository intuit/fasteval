# Configuration

## Overview

Three mechanisms control plugin behavior, listed by precedence: decorator parameters (highest), code-based configuration via `configure_langfuse()`, and environment variables (lowest). Most teams use environment variables for credentials and code config for operational defaults, then override per-test with decorator parameters when needed.

## Authentication

### Required Credentials

Connecting to Langfuse requires two API keys: a **Public Key** (`LANGFUSE_PUBLIC_KEY`, starts with `pk-lf-`) that identifies your project, and a **Secret Key** (`LANGFUSE_SECRET_KEY`, starts with `sk-lf-`) that authenticates requests.

To get these, log into your Langfuse dashboard, go to **Settings** > **API Keys**, and create a new key pair or copy existing credentials. Secret keys can't be retrieved after creation, so store them safely.

### Environment Variables (Recommended)

For production and CI/CD, environment variables are the way to go:

```bash
# Required
export LANGFUSE_PUBLIC_KEY="pk-lf-..."
export LANGFUSE_SECRET_KEY="sk-lf-..."

# Optional (defaults to Langfuse Cloud)
export LANGFUSE_HOST="https://cloud.langfuse.com"
```

Don't commit credentials to version control. Use a secrets manager (AWS Secrets Manager, HashiCorp Vault, etc.) in production. Rotate keys periodically, and keep separate keys for dev, staging, and prod.

### Code-Based Configuration

When environment variables aren't practical:

```python
from fasteval_langfuse import configure_langfuse, LangfuseConfig

configure_langfuse(LangfuseConfig(
    public_key="pk-lf-...",
    secret_key="sk-lf-...",
    host="https://cloud.langfuse.com"
))
```

This works well for local development, dynamic configuration, integration tests with isolated projects, and apps managing multiple Langfuse connections.

### Self-Hosted Langfuse

Point to your instance URL:

```bash
export LANGFUSE_HOST="https://langfuse.your-company.com"
```

Or in code:

```python
configure_langfuse(LangfuseConfig(
    host="https://langfuse.your-company.com"
))
```

The plugin talks to Langfuse over its REST API, so make sure your instance is reachable from wherever you're running evaluations.

## Configuration Options

### LangfuseConfig Reference

```python
from fasteval_langfuse import LangfuseConfig

config = LangfuseConfig(
    # Authentication
    public_key: str = None,           # Langfuse public API key
    secret_key: str = None,           # Langfuse secret API key
    host: str = "https://cloud.langfuse.com",  # Langfuse instance URL

    # Defaults
    default_project: str = None,      # Default project for trace queries

    # Score Reporting
    auto_push_scores: bool = True,    # Automatically push scores to Langfuse
    score_name_prefix: str = "fasteval_",  # Prefix for score names

    # Performance
    batch_size: int = 50,             # Traces per API request
    max_parallel_evals: int = 5,      # Concurrent evaluation workers
    retry_on_failure: bool = True,    # Retry failed API calls
)
```

### Detailed Option Reference

#### `public_key` and `secret_key`

Your Langfuse API credentials. Falls back to `LANGFUSE_PUBLIC_KEY` and `LANGFUSE_SECRET_KEY` environment variables if not set.

```python
# Explicit credentials
LangfuseConfig(
    public_key="pk-lf-1234567890abcdef",
    secret_key="sk-lf-abcdef1234567890"
)

# Or rely on environment variables
LangfuseConfig()  # Reads from LANGFUSE_PUBLIC_KEY, LANGFUSE_SECRET_KEY
```

#### `host`

The Langfuse instance URL. Defaults to Langfuse Cloud (`https://cloud.langfuse.com`).

```python
# Langfuse Cloud (default)
LangfuseConfig(host="https://cloud.langfuse.com")

# Self-hosted
LangfuseConfig(host="https://langfuse.internal.company.com")

# Local development
LangfuseConfig(host="http://localhost:3000")
```

#### `default_project`

Default project name when `@langfuse_traces` doesn't specify one. Keeps your decorator syntax shorter:

```python
configure_langfuse(LangfuseConfig(default_project="production"))

# Now project is optional:
@langfuse_traces(filter_tags=["customer-support"])  # Uses "production"
def test_traces(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

#### `auto_push_scores`

When `True` (the default), scores show up in the Langfuse UI right after evaluation. Set to `False` for dry runs, dev/debugging sessions where you don't want test scores cluttering Langfuse, or when you've got custom reporting logic.

```python
# Disable globally
LangfuseConfig(auto_push_scores=False)

# Or disable per-test
@langfuse_traces(project="prod", auto_push_scores=False)
def test_dry_run(trace_id, input, output, context, metadata):
    fe.score(output, input=input)  # Evaluates but doesn't push scores
```

#### `score_name_prefix`

Applied to all score names when pushing to Langfuse. Helps you tell fasteval scores apart from other scoring systems.

```python
# Default prefix
LangfuseConfig(score_name_prefix="fasteval_")
# Creates: fasteval_correctness, fasteval_relevance, fasteval_aggregate

# Custom prefix
LangfuseConfig(score_name_prefix="eval_v2_")
# Creates: eval_v2_correctness, eval_v2_relevance, eval_v2_aggregate

# No prefix
LangfuseConfig(score_name_prefix="")
# Creates: correctness, relevance, aggregate
```

**Tip:** A prefix makes filtering easy in the Langfuse UI: `score.fasteval_* < 0.8`

#### `batch_size`

Traces fetched per API request. Lower values (e.g., 20) use less memory but make more API calls. Higher values (e.g., 100) fetch faster but need more memory.

```python
# Default
LangfuseConfig(batch_size=50)

# Memory-constrained environment
LangfuseConfig(batch_size=20)

# Fast network, plenty of memory
LangfuseConfig(batch_size=100)
```

#### `max_parallel_evals`

Concurrent evaluation workers. Each one makes an LLM judge call. Lower values (e.g., 2) are safer for rate-limited providers; higher values (e.g., 10) speed things up.

```python
# Default
LangfuseConfig(max_parallel_evals=5)

# Rate-limited LLM provider
LangfuseConfig(max_parallel_evals=2)

# High-throughput environment
LangfuseConfig(max_parallel_evals=10)
```

#### `retry_on_failure`

Retries failed Langfuse API calls (fetching traces, pushing scores). Set to `False` if you want to fail fast during debugging.

```python
# Default: retry on transient failures
LangfuseConfig(retry_on_failure=True)

# Fail fast (useful for debugging)
LangfuseConfig(retry_on_failure=False)
```

## Environment-Specific Setup

### Development

Use a separate project to keep test data out of production:

```python
# conftest.py or test setup
import os
from fasteval_langfuse import configure_langfuse, LangfuseConfig

if os.getenv("ENV") != "production":
    configure_langfuse(LangfuseConfig(
        default_project="development",
        auto_push_scores=False,  # Don't push to Langfuse during dev
    ))
```

### CI/CD Pipelines

Pull credentials from secrets management:

```yaml
# GitHub Actions example
name: Evaluation
on:
  schedule:
    - cron: '0 */6 * * *'  # Every 6 hours

jobs:
  evaluate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: pip install fasteval-core fasteval-langfuse

      - name: Run evaluations
        env:
          LANGFUSE_PUBLIC_KEY: ${{ secrets.LANGFUSE_PUBLIC_KEY }}
          LANGFUSE_SECRET_KEY: ${{ secrets.LANGFUSE_SECRET_KEY }}
        run: pytest tests/evaluation/ -v
```

### Production Monitoring

Prioritize reliability and throughput:

```python
configure_langfuse(LangfuseConfig(
    default_project="production",
    auto_push_scores=True,
    batch_size=100,
    max_parallel_evals=10,
    retry_on_failure=True,
    score_name_prefix="fasteval_prod_",
))
```

### Multiple Environments

Switch between environments with a single variable:

```python
import os
from fasteval_langfuse import configure_langfuse, LangfuseConfig

env = os.getenv("ENVIRONMENT", "development")

configure_langfuse(LangfuseConfig(
    default_project=env,
    auto_push_scores=(env == "production"),
    score_name_prefix=f"fasteval_{env}_",
))
```

```bash
ENVIRONMENT=staging pytest tests/evaluation/
ENVIRONMENT=production pytest tests/evaluation/
```

## Advanced Configuration

### Per-Test Overrides

Decorator parameters override global config for individual tests:

```python
# Global config: auto_push_scores=True, default_project="production"

@langfuse_traces(
    project="staging",          # Override project
    auto_push_scores=False      # Override score pushing
)
def test_staging_dry_run(trace_id, input, output, context, metadata):
    fe.score(output, input=input)
```

### Custom Client Instance

For advanced scenarios, create and use a custom Langfuse client:

```python
from fasteval_langfuse import LangfuseClient
from fasteval_langfuse.trace_fetcher import TraceFetcher
from fasteval_langfuse.score_reporter import ScoreReporter

# Custom client with explicit credentials
client = LangfuseClient(
    public_key="pk-lf-...",
    secret_key="sk-lf-...",
    host="https://custom.langfuse.com"
)

# Use in components
fetcher = TraceFetcher(client=client)
traces, total = fetcher.fetch_and_sample(
    project="prod",
    filter_tags=["customer-support"],
    time_range="last_24h"
)
```

### Configuration Precedence

When the same setting appears at multiple levels, higher precedence wins:

```
1. Decorator parameters     (highest)
2. configure_langfuse()
3. Environment variables
4. Built-in defaults        (lowest)
```

Example:

```python
# Environment: LANGFUSE_PUBLIC_KEY=pk-env

configure_langfuse(LangfuseConfig(
    default_project="global-default"
))

@langfuse_traces(project="decorator-override")  # This wins
def test_traces(trace_id, input, output, context, metadata):
    pass
```

## Troubleshooting

### Authentication Errors

**Error:** `ValueError: Langfuse credentials required`

Missing or invalid API keys. Verify your environment variables are set (`echo $LANGFUSE_PUBLIC_KEY`), check for typos, confirm the keys belong to the right project, and double-check code config if you're using it.

### Connection Errors

**Error:** `ConnectionError: Unable to connect to Langfuse`

Network issue or wrong host URL. Verify `LANGFUSE_HOST`, check connectivity, make sure self-hosted instances are up and reachable, and look for proxy or firewall problems.

### Empty Trace Results

Evaluation runs but finds zero traces. Usually this means the project name doesn't match exactly (it's case-sensitive), filters are too narrow, or traces haven't flushed to Langfuse yet. Try widening the time range to `last_7d`, removing tag filters temporarily, and checking the Langfuse UI to confirm traces exist.

### Score Push Failures

Evaluation finishes but scores don't show up in Langfuse. Check that `auto_push_scores` isn't set to `False`, enable `retry_on_failure=True`, verify the Langfuse API is healthy, and look at pytest output for error messages.
