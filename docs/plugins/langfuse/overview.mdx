# Langfuse Plugin Overview

## Introduction

The **fasteval-langfuse** plugin lets you evaluate real production traces from [Langfuse](https://langfuse.com) using fasteval's LLM-as-judge metrics. You're already capturing traces. This plugin scores them automatically and pushes results back to Langfuse so you can see quality data right next to your operational data.

It's built for teams that use Langfuse for observability and want to go beyond eyeballing traces — automated evaluation on a schedule, with scores that actually mean something.

## The Evaluation Gap

You've instrumented your app. You can see every prompt and every response in Langfuse. But visibility alone doesn't tell you whether the responses are any good.

Manual review doesn't scale past a few hundred traces a day. Synthetic test suites can't capture the weird stuff real users do — slang, typos, follow-up questions out of nowhere. And periodic offline evaluations create a weeks-long delay between "something broke" and "we noticed."

The fix is straightforward: evaluate production data continuously. Your traces already exist in Langfuse. The plugin runs evaluations on a schedule (or on-demand via pytest), pushes scores back to each trace, and uses statistical sampling to keep costs under control. Quality issues show up in hours, not weeks.

## How It Works

The plugin follows a five-step workflow:

![Langfuse Integration Architecture](/docs/assets/langfuse-architecture.svg)

### Step 1: Trace Collection (Existing Infrastructure)

Your app already sends traces to Langfuse via the SDK. Each trace captures the input, response, any retrieved context, and custom metadata. Nothing changes here — the plugin works with what you've already got.

### Step 2: Trace Fetching

When you run an evaluation test, the `@langfuse_traces` decorator connects to Langfuse's API and fetches traces matching your criteria. You can filter by project, tags, time range, or specific users/sessions.

### Step 3: Intelligent Sampling

Evaluating every trace gets expensive fast. The plugin ships with five sampling strategies — random, stratified, score-based, recent-first, and no-sampling — so you can evaluate a representative subset without blowing your budget.

### Step 4: Metric Evaluation

Each sampled trace gets evaluated using fasteval's LLM-as-judge metrics. These assess things like correctness, faithfulness, relevance, and hallucination. The evaluation runs locally in your test environment, so you control which LLM provider does the judging.

### Step 5: Score Synchronization

After evaluation, scores are pushed back to each trace in Langfuse automatically. From there you can filter traces by quality, build dashboards tracking trends over time, and set up alerts when scores drop.

## Core Concepts

### Traces vs. Datasets

The plugin supports two evaluation patterns:

**Trace Evaluation** (`@langfuse_traces`) evaluates real production data. Use it for continuous monitoring, debugging live issues, and understanding how users actually interact with your app.

**Dataset Evaluation** (`@langfuse_dataset`) evaluates against curated test sets stored in Langfuse. Use it for regression testing, A/B testing prompts or models, and benchmarking against golden references.

### The Decorator Pattern

Both decorators wrap a test function that receives data as parameters. When the test runs, the decorator fetches traces or dataset items, calls your function once per item, collects results, and reports outcomes through pytest (and optionally back to Langfuse).

This fits naturally into existing pytest workflows and CI/CD pipelines.

### Automatic Score Reporting

By default, evaluation results get pushed back to Langfuse as scores on each trace:

```
After evaluation, each trace in Langfuse displays:
├── fasteval_correctness: 0.87
├── fasteval_faithfulness: 0.92
├── fasteval_relevance: 0.85
└── fasteval_aggregate: 0.88
```

You can configure the score name prefix, disable automatic reporting, or implement custom reporting logic.

## Installation

### Requirements

- Python 3.9 or higher
- A Langfuse account (cloud or self-hosted)
- API credentials from your Langfuse project

### Install the Package

Install fasteval-langfuse alongside the core fasteval library:

```bash
pip install fasteval-core fasteval-langfuse
```

### Configure Credentials

The simplest approach uses environment variables:

```bash
export LANGFUSE_PUBLIC_KEY="pk-lf-..."
export LANGFUSE_SECRET_KEY="sk-lf-..."
export LANGFUSE_HOST="https://cloud.langfuse.com"
```

For self-hosted Langfuse instances, set `LANGFUSE_HOST` to your deployment URL.

Alternatively, configure credentials in code:

```python
from fasteval_langfuse import configure_langfuse, LangfuseConfig

configure_langfuse(LangfuseConfig(
    public_key="pk-lf-...",
    secret_key="sk-lf-...",
    host="https://cloud.langfuse.com"
))
```

See [Configuration](/docs/plugins/langfuse/configuration) for complete options.

## Quick Start

### Evaluate Production Traces

Create a test file that evaluates customer support traces from the last 24 hours:

```python
# test_production.py
from fasteval_langfuse import langfuse_traces
from fasteval_langfuse.sampling import RandomSamplingStrategy
import fasteval as fe

@fe.correctness(threshold=0.8)
@fe.hallucination(threshold=0.9)
@langfuse_traces(
    project="production",
    filter_tags=["customer-support"],
    time_range="last_24h",
    sampling=RandomSamplingStrategy(sample_size=200)
)
def test_production_quality(trace_id, input, output, context, metadata):
    """
    Evaluate 200 random traces from customer support interactions.

    Parameters are populated automatically from each trace:
    - trace_id: Unique identifier for score reporting
    - input: The user's query
    - output: The LLM's response
    - context: Retrieved documents (if RAG application)
    - metadata: Custom fields stored with the trace
    """
    fe.score(output, input=input)
```

Run with pytest:

```bash
pytest test_production.py -v
```

**Expected Output:**

```
========================= test session starts ==========================
collecting ...
Evaluating 200/5,432 traces (3.7% sample, strategy=RandomSamplingStrategy)

test_production.py::test_production_quality PASSED

Evaluation Results:
  ✓ correctness: 0.87 (threshold: 0.80)
  ✓ hallucination: 0.94 (threshold: 0.90)
  Aggregate Score: 0.905

Scores pushed to 200 traces in Langfuse
========================= 1 passed in 45.23s ===========================
```

### Evaluate a Golden Dataset

Create a test that validates your agent against a curated dataset:

```python
# test_regression.py
from fasteval_langfuse import langfuse_dataset
import fasteval as fe

@fe.correctness(threshold=0.85)
@fe.relevance(threshold=0.8)
@langfuse_dataset(name="qa-golden-set", version="v2")
def test_golden_dataset(input, expected_output):
    """
    Validate agent against golden reference answers.

    Parameters come from dataset item fields:
    - input: The test query
    - expected_output: The reference answer to compare against
    """
    response = my_agent(input)
    fe.score(response, expected_output, input=input)
```

## When to Use This Plugin

### Ideal Use Cases

**Continuous Production Monitoring** — Run hourly or daily evaluations on a sample of production traces. Catch quality degradation before users complain.

**Root Cause Analysis** — When user feedback flags issues, evaluate all traces from affected users or time periods to find patterns.

**Pre-Deployment Validation** — Test new agent versions against golden datasets before shipping.

**A/B Testing** — Compare prompts, models, or configurations using the same dataset.

### When to Consider Alternatives

**Real-time evaluation** — This plugin does batch evaluation. For scoring during request processing, use fasteval's core library with direct integration.

**Very high volume** — If you need to evaluate millions of traces daily, sampling helps, but you may need custom infrastructure at that scale.

**Non-Langfuse observability** — This plugin is Langfuse-specific. For other platforms, use fasteval's core library with custom data loading.
