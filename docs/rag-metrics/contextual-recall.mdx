# Contextual Recall Metric

So your retriever found 5 highly relevant documents -- great precision. But did those documents actually contain everything needed to answer the question? That's what contextual recall measures: information coverage.

**Put simply:** "Did we retrieve all the facts we need?"

## Quick Example

```python
import fasteval as fe

@fe.contextual_recall(threshold=0.8)
def test_information_coverage():
    query = "What are the benefits of exercise?"
    expected = "Exercise improves cardiovascular health, mental well-being, and weight management."

    docs = retriever.search(query)
    response = rag_agent(query, context=docs)

    fe.score(response, expected, retrieval_context=docs, input=query)
```

## Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `threshold` | float | 0.7 | Minimum score to pass (0.0 - 1.0) |
| `model` | str | None | Override the default LLM model |

## How Scoring Works

**Formula:** `facts_found_in_context / total_facts_in_expected`

| Score | Interpretation |
|-------|---------------|
| **1.0** | All needed facts are in retrieved docs |
| **0.75** | 3 of 4 needed facts found |
| **0.5** | Half the needed info missing |
| **0.0** | None of the needed info retrieved |

## Practical Examples

### Knowledge Base Coverage Testing

Does your knowledge base actually contain the answers?

```python
@fe.contextual_recall(threshold=0.85)
@fe.csv("golden_qa_pairs.csv")
def test_kb_has_answers(query, expected_answer):
    """Verify knowledge base covers all expected information."""
    docs = knowledge_base.search(query)
    response = rag_agent(query, context=docs)

    fe.score(response, expected_answer, retrieval_context=docs, input=query)
```

### Identifying Knowledge Gaps

If this fails, you know exactly which specs are missing from your docs.

## Precision vs Recall Trade-off

These metrics often pull in opposite directions:

| Scenario | Precision | Recall | Cause |
|----------|-----------|--------|-------|
| Retrieve few, highly relevant docs | High | Low | Missing information |
| Retrieve many docs broadly | Low | High | Lots of noise |
| Well-tuned retriever | Balanced | Balanced | Sweet spot |

Always test both:

```python
@fe.contextual_precision(threshold=0.7)
@fe.contextual_recall(threshold=0.8)
def test_balanced_retrieval():
    docs = retriever.search(query)
    fe.score(response, expected, retrieval_context=docs, input=query)
```

## Choosing Your Threshold

| Use Case | Suggested Threshold |
|----------|---------------------|
| Thorough answers needed | 0.9+ |
| General Q&A | 0.7 - 0.8 |
| Acceptable partial answers | 0.5 - 0.7 |

## Diagnosing Low Recall

When recall is low, work through these questions in order.

**Is the information in your knowledge base at all?** Check whether the expected facts exist in any document. You might just need to add more content.

**Is chunking splitting important information?** An expected fact might span two chunks. Try larger chunk sizes or add chunk overlap.

**Is k too low?** The relevant doc might sit at position 6 when you're only pulling back 5. Bump k up and see if the score improves.

**Is the embedding model finding semantic matches?** Sometimes the query phrasing doesn't match document phrasing well enough. Consider hybrid search (semantic + keyword) to cover both angles.

## Best Practices

1. **Always provide expected_output** -- recall needs ground truth to compare against.
2. **Use realistic expected answers** that reflect what a good answer actually looks like.
3. **Combine with precision** -- high recall plus low precision means noisy retrieval.
4. **Test edge cases** -- especially complex queries that need information from multiple docs.
