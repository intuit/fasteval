# Answer Correctness Metric

Is your RAG system's answer actually right? That's what this metric tells you. It goes beyond surface-level similarity and does a claim-by-claim factual comparison -- identifying what's correct, what's missing, and what's flat-out wrong.

## Quick Example

```python
import fasteval as fe

@fe.answer_correctness(threshold=0.8)
def test_rag_accuracy():
    query = "When was the Eiffel Tower built?"
    expected = "The Eiffel Tower was built in 1889."

    docs = retriever.search(query)
    response = rag_agent(query, context=docs)

    fe.score(response, expected, context=docs, input=query)
```

## Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `threshold` | float | 0.7 | Minimum score to pass (0.0 - 1.0) |
| `model` | str | None | Override the default LLM model |

## How It Works

Answer correctness breaks down claims and classifies each one:

```
Expected: "Python was created in 1991 by Guido van Rossum in the Netherlands."

Actual: "Python was created in 1991 by Guido van Rossum. It's open source."
```

**Analysis:**
- **True Positive:** "created in 1991" (matches expected)
- **True Positive:** "by Guido van Rossum" (matches expected)
- **False Negative:** "in the Netherlands" (missing from actual)
- **Extra Info:** "open source" (not in expected, but factually true)

**Score calculation weighs these factors**, penalizing contradictions heavily and missing info moderately.

## Practical Examples

### Golden Dataset Evaluation

Test your RAG against a curated set of correct answers:

```python
@fe.answer_correctness(threshold=0.8)
@fe.csv("golden_qa_dataset.csv")
def test_against_golden_set(query, correct_answer):
    docs = retriever.search(query)
    response = rag_agent(query, context=docs)

    fe.score(response, correct_answer, context=docs, input=query)
```

### Detecting Contradictions

This metric specifically catches conflicting information:

```python
@fe.answer_correctness(threshold=0.9)
def test_no_contradictions():
    expected = "Our return policy is 30 days."

    docs = retriever.search("return policy")
    response = rag_agent("What's the return policy?", context=docs)

    # Will fail if response says "60 days" or "no returns"
    fe.score(response, expected, context=docs)
```

## Answer Correctness vs Correctness

| Metric | Analysis Type | Best For |
|--------|--------------|----------|
| `answer_correctness` | Claim-by-claim factual comparison | RAG with ground truth |
| `correctness` | Overall semantic similarity | General LLM outputs |

Reach for `answer_correctness` when you've got "golden" correct answers and factual precision matters -- think legal, medical, or financial use cases where contradictions are deal-breakers. It's also the go-to for measuring overall RAG system quality.

`correctness` works better when semantic equivalence is enough, you don't have ground truth, or you're evaluating creative and open-ended outputs.

## Complete RAG Evaluation Stack

For production RAG systems, we recommend testing multiple dimensions:

```python
@fe.faithfulness(threshold=0.85)          # Grounded in context?
@fe.contextual_precision(threshold=0.7)   # Retrieved docs relevant?
@fe.contextual_recall(threshold=0.75)     # All needed info retrieved?
@fe.answer_correctness(threshold=0.8)     # Final answer correct?
def test_production_rag():
    query = "What are the symptoms of flu?"
    expected = "Flu symptoms include fever, cough, body aches, and fatigue."

    docs = retriever.search(query)
    response = rag_agent(query, context=docs)

    fe.score(response, expected, context=docs, input=query)
```

## Choosing Your Threshold

| Use Case | Suggested Threshold |
|----------|---------------------|
| Medical/legal/financial | 0.9+ |
| Product documentation | 0.8 - 0.9 |
| General knowledge Q&A | 0.7 - 0.8 |
| Conversational with tolerance | 0.6 - 0.7 |

## Best Practices

1. **Write precise expected answers** -- be specific about the facts that matter.
2. **Include all key facts** -- anything missing from the expected answer won't get checked.
3. **Test with varied phrasings** -- the same fact can be expressed many ways.
4. **Combine with faithfulness** -- a correct answer still needs to be grounded in context.
