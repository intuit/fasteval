# Coherence Metric

Does the response actually make sense? Coherence measures whether ideas connect naturally, the structure holds together, and the language reads clearly. It's the difference between a well-organized explanation and a jumbled mess of loosely related sentences.

## Quick Example

```python
import fasteval as fe

@fe.coherence(threshold=0.8)
def test_explanation():
    response = my_llm("Explain how photosynthesis works")
    fe.score(response)
```

## Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `threshold` | float | 0.8 | Minimum score to pass (0.0 - 1.0) |
| `model` | str | None | Override the default LLM model |

## How Scoring Works

The LLM judge evaluates multiple aspects of coherence:

| Score | Interpretation | Example |
|-------|---------------|---------|
| **0.9-1.0** | Perfectly coherent | Flawless flow, structure, and clarity |
| **0.7-0.9** | Very coherent | Well-organized with minor issues |
| **0.5-0.7** | Moderately coherent | Some flow or clarity problems |
| **0.0-0.5** | Incoherent | Disjointed, confusing, or unreadable |

### What Gets Evaluated

The judge looks at logical flow — do ideas connect naturally from one to the next? It checks structure: is there a clear progression, or does the response jump around randomly? Clarity and grammar matter too. And it checks whether the tone stays consistent throughout, since a response that shifts between casual and academic mid-paragraph feels off even if the facts are right.

## Practical Examples

### Technical Explanations

```python
@fe.coherence(threshold=0.85)
def test_technical_clarity():
    response = my_llm("Explain the difference between REST and GraphQL APIs")
    fe.score(response)
```

### Multi-Step Instructions

```python
@fe.coherence(threshold=0.8)
def test_instructions():
    response = my_llm("How do I set up a Python virtual environment?")
    fe.score(response)
```

### Long-Form Content

```python
@fe.coherence(threshold=0.75)
def test_blog_post():
    response = my_llm("Write a blog post introduction about AI in healthcare")
    fe.score(response)
```

## Choosing Your Threshold

Quick answers can get away with 0.7-0.75. General explanations should target 0.75-0.8. Technical documentation needs 0.8-0.85, and anything you're publishing should hit 0.85 or above.

## When to Use Coherence

Coherence shines for educational content, technical explanations, multi-step instructions — basically anything where readability matters. If you need to check factual accuracy instead, reach for `@fe.correctness()`. For coverage of a topic, use `@fe.completeness()`. And for specific formatting requirements, `@fe.instruction_following()` is the better fit.

## Combining with Other Metrics

For well-rounded quality evaluation:

```python
@fe.coherence(threshold=0.8)      # Is it well-written?
@fe.correctness(threshold=0.8)    # Is it accurate?
@fe.completeness(threshold=0.7)   # Does it cover the topic?
def test_educational_content():
    question = "Explain machine learning to a beginner"
    response = my_llm(question)
    fe.score(response, input=question)
```

What we've found is that stacking these three gives you a solid baseline for any educational or explanatory use case.
