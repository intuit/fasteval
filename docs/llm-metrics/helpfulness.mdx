# Helpfulness Metric

A response can be correct, coherent, and complete — and still useless. Helpfulness measures whether the response actually helps someone accomplish their goal. Does it give actionable, practical information, or just restate the question in fancier words?

## Quick Example

```python
import fasteval as fe

@fe.helpfulness(threshold=0.7)
def test_useful_response():
    query = "How do I fix a leaky faucet?"
    response = my_llm(query)
    fe.score(response, input=query)
```

## Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `threshold` | float | 0.7 | Minimum score to pass (0.0 - 1.0) |
| `model` | str | None | Override the default LLM model |

## How Scoring Works

The LLM judge evaluates practical utility:

| Score | Interpretation | Example |
|-------|---------------|---------|
| **0.9-1.0** | Extremely helpful | Actionable, practical, anticipates needs |
| **0.7-0.9** | Very helpful | Good practical advice with minor gaps |
| **0.5-0.7** | Somewhat helpful | Useful but could be more practical |
| **0.0-0.5** | Not helpful | Theoretical only or misses the point |

### What Gets Evaluated

The judge looks at whether the response provides actionable information — things the user can actually do. It checks practical value (would this genuinely help?), applicability (is the advice relevant to their situation?), and whether it anticipates follow-up needs. The level of detail matters too. Too shallow and it's useless; too deep and it's overwhelming.

## Practical Examples

### How-To Questions

```python
@fe.helpfulness(threshold=0.8)
def test_troubleshooting():
    query = "My Python script is running slowly, how can I optimize it?"
    response = my_llm(query)
    fe.score(response, input=query)
```

### Decision Support

```python
@fe.helpfulness(threshold=0.75)
def test_recommendation():
    query = "Should I use PostgreSQL or MongoDB for my e-commerce app?"
    response = my_llm(query)
    fe.score(response, input=query)
```

### Customer Support Scenarios

```python
@fe.helpfulness(threshold=0.85)
def test_support_response():
    query = "I can't log into my account and the password reset isn't working"
    response = my_llm(query)
    fe.score(response, input=query)
```

## Choosing Your Threshold

General Q&A is fine at 0.6-0.7. Technical support should hit 0.7-0.8 since users come with specific problems. Customer-facing responses need 0.8-0.85 — a bad support answer is worse than no answer. For critical support scenarios, aim for 0.85 or above.

## When to Use Helpfulness

Helpfulness is the right metric for how-to questions, troubleshooting guides, customer support responses, recommendation queries, and decision support. If you're checking factual accuracy, use `@fe.correctness()`. For educational depth, `@fe.completeness()` is a better fit. Safety concerns call for `@fe.toxicity()` or `@fe.bias()`.

## Combining with Other Metrics

For customer-facing applications:

```python
@fe.helpfulness(threshold=0.8)    # Is it useful?
@fe.correctness(threshold=0.9)    # Is it accurate?
@fe.toxicity(threshold=0.95)      # Is it safe?
def test_customer_response():
    query = "How do I cancel my subscription?"
    response = my_llm(query)
    fe.score(response, input=query)
```

For technical support:

```python
@fe.helpfulness(threshold=0.8)    # Is it practical?
@fe.completeness(threshold=0.7)   # Does it cover the steps?
@fe.coherence(threshold=0.8)      # Is it easy to follow?
def test_tech_support():
    query = "How do I configure SSL for my server?"
    response = my_llm(query)
    fe.score(response, input=query)
```

In practice, helpfulness paired with correctness is the most common stack we see for support use cases. You want answers that are both right and actually useful.
