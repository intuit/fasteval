# Human Review

Some things automated metrics just can't judge. The `@fe.human_review` decorator drops an interactive prompt into your test run so a human reviewer can score responses on the spot — perfect for subjective quality assessments.

![Human Review Flow](docs/assets/human-review-flow.svg)

## Basic Usage

```python
import fasteval as fe

@fe.human_review(prompt="Is this response helpful and accurate?")
@fe.correctness(threshold=0.8)
def test_qa():
    response = agent(query)
    fe.score(response, expected, input=query)
```

## Scoring Options

Reviewers can enter:

| Input | Score | Description |
|-------|-------|-------------|
| `1` | 0.00 | Poor |
| `2` | 0.25 | Below average |
| `3` | 0.50 | Average |
| `4` | 0.75 | Good |
| `5` | 1.00 | Excellent |
| `p` or `pass` | 1.00 | Pass |
| `f` or `fail` | 0.00 | Fail |
| `s` or `skip` | - | Skip review |

## Multi-Turn Conversations

For conversations, the full history is displayed:

```python
@fe.human_review(prompt="Is context maintained correctly?")
@fe.context_retention(threshold=0.8)
@fe.conversation([
    {"query": "My name is Alice"},
    {"query": "What's my name?", "expected": "Alice"},
])
def test_memory(query, expected, history):
    response = agent(query, history=history)
    fe.score(response, expected, input=query, history=history)
```

## Configuration Options

```python
@fe.human_review(
    prompt="Is this response accurate?",  # Question to ask
    required=False,                        # If True, skip fails the test
    threshold=0.5,                         # Minimum score to pass
)
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `prompt` | str | "Please review this response" | Question shown to reviewer |
| `required` | bool | False | If True, skipping fails the test |
| `threshold` | float | 0.5 | Minimum human score to pass |

## CI/CD Mode

In automated pipelines, human review gets automatically skipped. You can also disable it explicitly.

### Using pytest flag

```bash
pytest tests/ --no-interactive
```

### Using environment variable

```bash
FASTEVAL_NO_INTERACTIVE=1 pytest tests/
```

### Programmatic check

```python
from fasteval.utils import is_interactive

if is_interactive():
    print("Running in interactive mode")
else:
    print("Running in CI/CD mode - human review skipped")
```

## Accessing Human Review Results

```python
@fe.human_review(prompt="Is this good?")
@fe.correctness(threshold=0.8)
def test_with_human_review():
    response = agent(query)
    result = fe.score(response, expected)

    # Access human review data
    if result.human_review:
        print(f"Human score: {result.human_review.score}")
        print(f"Raw input: {result.human_review.raw_input}")
        print(f"Skipped: {result.human_review.skipped}")
```

## Best Practices

Write specific prompts. "Is this response factually accurate?" tells the reviewer exactly what to look for — "Is this good?" doesn't. We recommend using human review to validate edge cases rather than replace automated metrics; it's a complement, not a substitute. Set higher thresholds for critical evaluations where quality really matters. And unless you want CI builds to fail when reviews are skipped, keep `required=False`.
